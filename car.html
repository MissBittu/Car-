PHANTOM STRIKE

An interactive automotive configurator featuring real-time generative AI suggestions, multi-view 3D showcase, dynamic pricing, and complete build submission. Built as a web application delivering game-engine-quality interaction via HTML5, SVG, CSS3, and the Claude AI API.

Game Mechanics

  • Click/select color, wheels, accent, environment
  • Drag to rotate car (side view)
  • Hover parts for live spec tooltips
  • 4 distinct camera views (Side/Front/Rear/Top)
  • Real-time price updates on every change
  • Build submission with confirmation modal
  • Reset to default functionality
  • JSON config copy to clipboard

Graphics

  • Full 3D garage environment in CSS perspective
  • Ceiling spotlights with cone rays
  • Perspective floor tile grid
  • Ambient car ground glow (animated)
  • Distinct SVG car per view angle
  • Distinct wheel spoke patterns per style
  • Caliper & accent color live on car
  • Drag-to-rotate wheel animation

Generative AI

  • Claude Sonnet API integration
  • 5 themes: Sporty / Luxury / Stealth / Racing / Off-Road
  • Generates 3–5 named builds per theme
  • AI "thinking" animation while generating
  • One-click apply to live configurator
  • JSON output with ai_theme field
  • Robust fallback config set
  • Each config includes desc + rationale

HOW TO INTERACT

  • Select a theme (Sporty / Luxury / Stealth / Racing / Off-Road)
  • Click ⚡ GENERATE AI BUILDS — Claude returns 3 named configurations
  • Click any AI card to instantly apply it to the car
  • Manually refine: pick a color swatch, wheel style, accent finish, or environment
  • Hover over car parts in Side view to see component specs
  • Drag left/right on the car to rotate the wheels
  • Switch views: SIDE → FRONT → REAR → TOP
  • Click SUBMIT FINAL BUILD to confirm and copy your JSON config

AI COMPONENT EXPLANATION

The AI integration uses Anthropic's Claude Sonnet model via the /v1/messages REST API. A structured system prompt constrains the model to return valid JSON arrays of car configurations. Each theme input produces a distinct set of builds with unique color, wheel, accent, and a descriptive tagline.

POST /v1/messages System: "Return exactly 3-5 car configs as JSON array. Fields: name, color, wheel, accent, desc, rationale." User: "Theme: sporty. Generate builds for the Phantom Strike." → Claude returns structured JSON → parsed → rendered as clickable cards → one-click applies to live car

If the API is unavailable (network, CORS), the system falls back to curated pre-built configurations ensuring the demo always works.
